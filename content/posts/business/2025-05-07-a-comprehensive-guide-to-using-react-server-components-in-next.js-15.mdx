## Introduction to React Server Components

React Server Components are designed to render on the server, allowing you to offload heavy computations and data fetching to the server-side. This results in faster load times and improved performance, as the client receives a pre-rendered HTML page.

### Key Benefits

- **Improved Performance**: By rendering components on the server, you reduce the amount of JavaScript sent to the client.
- **SEO Friendly**: Server-rendered pages are more easily indexed by search engines.
- **Simplified Data Fetching**: Fetch data directly in server components without worrying about client-side data fetching libraries.

## Creating Your First Server Component

In Next.js 15, server components are created by default. To create a server component, simply create a new file in the `pages` directory.

```jsx
// pages/ServerComponentExample.js

export default function ServerComponentExample() {
  return (
    <div>
      <h1>Hello from Server Component</h1>
    </div>
  );
}
```

This component will be rendered on the server and sent as HTML to the client.

## Fetching Data in Server Components

One of the main advantages of server components is the ability to fetch data directly on the server. You can use any data-fetching library, such as `axios` or the native `fetch` API.

```jsx
// pages/DataFetchingComponent.js

import React from 'react';

export default async function DataFetchingComponent() {
  const res = await fetch('https://api.example.com/data');
  const data = await res.json();

  return (
    <div>
      <h1>Data from Server</h1>
      <pre>{JSON.stringify(data, null, 2)}</pre>
    </div>
  );
}
```

## Combining Server and Client Components

In some cases, you may need to combine server and client components. Client components can handle interactivity and state management, while server components handle data fetching and rendering.

```jsx
// components/ClientComponent.js

'use client';

import { useState } from 'react';

export default function ClientComponent() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <button onClick={() => setCount(count + 1)}>Increment</button>
      <p>Count: {count}</p>
    </div>
  );
}

// pages/CombinedComponent.js

import ServerComponentExample from './ServerComponentExample';
import ClientComponent from '../components/ClientComponent';

export default function CombinedComponent() {
  return (
    <div>
      <ServerComponentExample />
      <ClientComponent />
    </div>
  );
}
```

## Best Practices

- **Minimize Client-Side JavaScript**: Use server components to reduce the amount of JavaScript sent to the client.
- **Optimize Data Fetching**: Fetch only the necessary data in server components to improve performance.
- **Use Client Components for Interactivity**: Reserve client components for parts of your application that require interactivity or state management.
